# Agent-First Product Patterns

Ramp Labs, Sierra, Cognition, Harvey, and Glean are useful benchmarks because they each show a different pattern Hermes can borrow:

- make agent behavior visible instead of magical
- package AI as named experiences, not vague “AI features”
- tie agents to workflows that save time for specific operator roles
- keep the public story grounded in productivity and concrete workflows
- expose enough surface area for users to see the agent doing work
- make the agent feel like a teammate, not a chat widget
- pair automation with governance and observability

For Hermes Mission Control, that suggests:
- a dashboard that shows work in motion
- agent launch and supervision controls
- visible automation steps and outcomes
- workflow-specific experiences instead of one generic chat box
- practical, business-facing AI features first
- domain-specific agent surfaces, not a single generalist blob
