# Agent-First Company Patterns

These companies are useful benchmarks because they show different ways to productize AI-first systems.

| Company | Core domain | Agent posture | Strongest lesson for Hermes |
|---|---|---|---|
| Ramp Labs | Finance/workflow research | Productized experiments | Make agent behavior visible and concrete |
| Sierra | Customer experience | Multi-channel customer agents | Treat comms as a first-class product surface |
| Cognition AI | Software engineering | Collaborative AI teammate | Give the agent a bounded, valuable job |
| Harvey AI | Legal/professional services | Domain-specific workflow agents | Ground agent output in source context and domain depth |
| Glean | Enterprise knowledge/work AI | Search + assistant + agents + governance | Combine retrieval, action, and governance in one system |

Shared patterns:
- agents are named product surfaces, not hidden internals
- the company story centers on concrete workflows
- observability and governance matter
- the product gets more useful when the agent has a specific job
- the UI is part of the product, not just a wrapper

What Hermes should steal:
- a visible operator cockpit
- domain-specific agent surfaces
- workflow-centered AI features
- governance and approvals built in
- one system that combines knowledge, action, and state
